# 微信云托管部署教程（零基础版）

## 🎯 部署方式：本地上传代码包

**优点**：不需要 GitHub，不需要 Docker，最简单！

---

## 📋 第一步：获取部署包

部署包已准备好，包含所有后端代码和配置文件：

```
server/
├── Dockerfile        # 构建配置
├── container.json    # 服务配置
├── .env              # 环境变量（已配置好）
├── src/              # 源代码
├── package.json      # 依赖配置
└── ...
```

---

## 📋 第二步：登录微信公众平台

1. 打开浏览器，访问：**https://mp.weixin.qq.com**
2. 使用小程序管理员微信扫码登录

---

## 📋 第三步：进入云托管

```
登录后 → 左侧菜单 → 云托管 → 服务管理
```

![位置示意]
```
┌─────────────────────────────────────┐
│  首页                               │
│  概况                               │
│  ─────────────────                  │
│  云托管          ← 点这里           │
│    ├─ 服务管理   ← 再点这里         │
│    ├─ 版本管理                      │
│    └─ ...                           │
└─────────────────────────────────────┘
```

---

## 📋 第四步：创建服务

1. 点击「**新建服务**」按钮
2. 填写信息：
   - **服务名称**：`octopus-broker`（或自定义）
   - **服务描述**：章鱼经纪人后端服务
3. 点击「确定」

---

## 📋 第五步：新建版本

1. 进入刚创建的服务
2. 点击「**新建版本**」
3. 选择「**本地上传**」或「**代码包上传**」

---

## 📋 第六步：上传代码包

1. 点击「上传代码包」
2. 选择 `server` 文件夹（或打包后的 zip/tar.gz）
3. 等待上传完成（约 1-2 分钟）

**或者**，如果支持文件夹选择：
- 直接选择项目的 `server` 文件夹上传

---

## 📋 第九步：发布服务

1. 点击「**发布**」按钮
2. 等待构建完成（约 3-5 分钟）
3. 服务状态变为「运行中」

---

## ✅ 验证部署成功

### 方法一：控制台查看
- 服务状态显示「运行中」
- 实例健康检查通过

### 方法二：接口测试
复制服务访问地址，在浏览器访问：
```
https://你的服务地址/api/hello
```

返回以下内容表示成功：
```json
{"code":200,"msg":"Hello from Octopus Broker!","data":null}
```

## 🆘 遇到问题？

### 问题1：找不到「云托管」菜单
- **原因**：需要先开通云托管服务
- **解决**：点击「开通」按钮，按提示完成开通

### 问题2：上传失败
- **原因**：文件太大或格式不对
- **解决**：确保上传的是 `server` 文件夹或其 zip 压缩包

### 问题3：构建失败
- **原因**：可能是环境变量未配置
- **解决**：检查第七步的环境变量是否都已配置
