## 1. 服务创建

### 1.1 基本信息

| 字段 | 填写内容 | 说明 |
|------|---------|------|
| **服务名称** | `octopus-broker` | 只能包含小写字母、数字和连字符，建议使用项目名 |
| **服务描述** | `章鱼经纪人后端服务 - 房产经纪人办公工具小程序API服务` | 简洁描述服务用途 |
| **所属环境** | 选择已创建的环境 | 如未创建，需先创建云托管环境 |

### 1.2 环境创建（如需新建）

| 字段 | 推荐配置 | 说明 |
|------|---------|------|
| **环境名称** | `prod` 或 `octopus-prod` | 生产环境标识 |
| **环境ID** | 自动生成或自定义 | 记住此ID，用于小程序配置 |


### 2.2 实例数量

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **最小实例数** | `1` | 避免冷启动，保持至少1个实例运行 |
| **最大实例数** | `5` | 根据业务峰值设置，初期可设小 |

### 2.3 配置建议

| 业务规模 | CPU | 内存 | 最小实例 | 最大实例 |
|---------|-----|------|---------|---------|
| 初期测试 | 0.25核 | 256MB | 0 | 3 |
| 小规模运营 | 0.5核 | 512MB | 1 | 5 |
| 中等规模 | 1核 | 1GB | 2 | 10 |
| 大规模 | 2核 | 2GB | 3 | 20 |

## 3. 扩缩容设置

### 3.1 扩缩容策略

```
┌─────────────────────────────────────────────────────────────┐
│  扩缩容策略                                                  │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  策略类型:  ● CPU使用率  ○ 内存使用率  ○ 请求QPS            │
│                                                              │
│  触发阈值:  [  80  ] %                                       │
│                                                              │
│  扩容冷却时间: [  60  ] 秒    （扩容后等待多久才能再次扩容）  │
│  缩容冷却时间: [ 300  ] 秒    （缩容后等待多久才能再次缩容）  │
│                                                              │
│  ⚠️ 缩容冷却时间建议设置较长，避免频繁缩容影响服务稳定性     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 各策略说明

| 策略类型 | 适用场景 | 推荐阈值 |
|---------|---------|---------|
| **CPU使用率** | CPU密集型应用 | 70-80% |
| **内存使用率** | 内存密集型应用 | 75-85% |
| **请求QPS** | IO密集型应用 | 根据业务评估 |

### 3.3 推荐配置

```json
{
  "policyType": "cpu",
  "policyThreshold": 80,
  "scaleUpCooldown": 60,
  "scaleDownCooldown": 300
}
```

### 4.2 健康检查说明

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **检查路径** | `/api/hello` | 后端已有的健康检查接口 |
| **检查间隔** | 30秒 | 不宜过短，避免频繁检查 |
| **超时时间** | 10秒 | 根据接口响应时间设置 |
| **启动延迟** | 30-60秒 | 需要大于应用启动时间 |

### 4.3 健康检查接口要求

后端接口 `/api/hello` 应返回：
```json
{
  "code": 200,
  "msg": "Hello from Octopus Broker!",
  "data": null
}
```

## 5. 网络配置

### 5.1 端口配置

```
┌─────────────────────────────────────────────────────────────┐
│  端口配置                                                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  容器端口:    [ 3000 ]                                       │
│                                                              │
│  协议:        ● HTTP  ○ HTTPS  ○ TCP  ○ UDP                 │
│                                                              │
│  ⚠️ 端口必须与 Dockerfile 中 EXPOSE 的端口一致              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 访问方式

| 访问方式 | 配置 | 说明 |
|---------|------|------|
| **公网访问** | ✅ 开启 | 允许公网通过域名访问 |
| **VPC内网访问** | ✅ 开启 | 允许同一VPC内服务互访 |
| **微信调用** | ✅ 开启 | 允许小程序通过云托管调用 |

### 5.3 域名配置

```
默认域名格式：
https://[服务名]-[环境ID].[区域].run.tcloudbase.com

示例：
https://octopus-broker-prod-xxx.sh.run.tcloudbase.com
```

**自定义域名配置**：
1. 在「访问设置」→「域名管理」中添加自定义域名
2. 添加 CNAME 解析到默认域名
3. 上传 SSL 证书（如需 HTTPS）

后端已配置 CORS，允许所有来源：
```typescript
app.enableCors({
  origin: true,
  credentials: true,
});
```

如需限制来源：
```typescript
app.enableCors({
  origin: ['https://your-domain.com'],
  credentials: true,
});
```

### 10.1 版本发布流程

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 构建 Docker 镜像                                        │

│  2. 推送到镜像仓库                                          │
│     docker push xxx/octopus-broker:v1.0.0                   │

│  3. 在云托管控制台创建新版本                                │
```

## 🆘 常见问题

### Q: 服务启动失败？

1. 检查健康检查路径是否正确
2. 增加启动延迟时间
3. 查看日志定位具体错误

### Q: 实例频繁扩缩容？

1. 调高扩缩容阈值
2. 增加冷却时间
3. 设置最小实例数为 1

### Q: 小程序调用超时？

1. 检查服务是否正常运行
2. 确认云托管环境 ID 配置正确
3. 检查小程序合法域名配置

### Q: 如何查看详细日志？

1. 进入「日志管理」
2. 选择时间范围
3. 输入关键词搜索
4. 可下载完整日志
